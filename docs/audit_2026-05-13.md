# Abundance Audit — Lookahead Bug & Methodology Gaps

**Date:** 2026-05-13  
**Status:** Critical — testnet positions should be paused until P0 items fixed

---

## Summary

The abundance crypto trading agent inflated Sharpe ratios by 3-10× via a classic one-bar lookahead bug. The adversarial validation pipeline is empty stubs — zero LLM critique was ever executed. Four strategies were approved that never passed a real review. Two still show genuine alpha after the fix (MA50, ADX-blend); two were pure lookahead artifacts.

---

## What Was Fixed (Verified ✓)

- **Lookahead bug fixed** in all 4 strategies: `sig[t] = 1.0 if close[t-1] > anchor[t-2] else 0.0` — decision on yesterday's data, earns today's return
- **Cost convention unified**: same `round_trip_cost` applied on signal transitions
- **Trade counts**: now computed from actual signal flips, not narrated
- **B&H baseline**: α = Sharpe(strat) − Sharpe(B&H) metric included
- **Donchian20**: correctly identified as lookahead casualty (Sharpe 3.91 → 0.36)

## What Is Still Broken

### 1. Lookahead Audit Is Theatre (P0)
`lookahead_check.py` re-derives the corrected formula inside the audit module and verifies its own rebuild against truncation. It never inspects the strategy's actual signal output. It would pass even if the strategies were reverted to the buggy version.
- **Fix:** Strategies must return their signal array; audit recomputes under data truncation at multiple `t` and asserts equality

### 2. Adversarial Pipeline Is Empty Stubs (P0)
`adversarial_node()`, `hypothesis_node()`, `coding_node()`, `decision_node()` all return empty strings. No LLM was ever called. The LangGraph workflow rubber-stamps everything.
- **Fix:** Wire adversarial node to call LLM with structured red-team checklist (Lopez de Prado ch. 11)
- Gate approval on severity ≤ low

### 3. Costs Are Double-Charged Per Round Trip (P0)
`abs(sig[i]-sig[i-1]) * round_trip_cost` charges a full round-trip on 0→1 entry AND again on 1→0 exit. A single round-trip costs 2× what it should.
- **Fix:** Charge `entry_cost` on entries (0→1), `exit_cost` on exits (1→0). Each ≈ half a round-trip.

### 4. Pull Testnet Positions (P0)
Positions opened with bugged signals. Restart paper trade from clean state after fixes.

### 5. No Deflated Sharpe Ratio (P1)
11 strategies tested, 2 kept. Selection inflates surviving Sharpes. DSR formula: with Sharpe ~1.1 selected from 11 trials over 2900 daily bars, deflated SR ≈ 0.75. Still positive but should be reported.
- **Reference:** Lopez de Prado, *Advances in Financial ML*, ch. 11

### 6. No Autocorrelation Adjustment in Sharpe (P1)
Trend strategies have positive serial correlation; standard √N annualization underestimates variance. Use Lo (2002) correction or Newey-West.
- **Reference:** Lo, A. W. (2002). "The Statistics of Sharpe Ratios"

### 7. Sharpes Are Full-Sample, Not Walk-Forward (P1)
Train params on 2017–2021, freeze, evaluate 2022–2025. Purged k-fold CV (LdP ch. 7).

### 8. Multi-Asset Confirmation Is One Bet (P1)
BTC/ETH correlation ~0.85. Test on uncorrelated regimes: SOL (different 2022 path due to FTX), BNB, non-crypto (FX, gold).

### 9. ADX-blend Has Lingering Issues (P2)
`ref` initialized at `close[0]` — ratchets from a price-history accident. Dead `discrete_sig` block (lines 100-105). Code was not reviewed.

### 10. MetricsReport.trades Is Monkey-Patched (P2)
Not a dataclass field, not in `to_dict()`, not in `print()`. Silently disappears on serialization.
- **Fix:** Add `trades` as a proper `MetricsReport` field

### 11. Strategies Don't Return Signal Arrays (P0)
`run_strategy()` returns `(eq_df, mc)` — no signal, no trades log, no indicator values. Debugging and auditing require these.
- **Fix:** Define strategy interface ABC with separable methods

### 12. EMA Computed on Full Series Then Indexed (P3)
Mathematically OK for EMA (one-step causal recursion) but sets dangerous pattern. Z-scores, rolling regressions, percentile transforms break under this idiom.
- **Fix:** Build indicators incrementally in a single causal loop, or use polars `rolling_*`
